1. OBJETIVO DE ESTA ETAPA
================================================================================

El objetivo de esta etapa fue dejar preparados los 8 archivos canonicos por fuente/editorial,
con la columna Area corregida y clasificada de forma mas consistente con el contenido real de
cada articulo.

En esta etapa NO se genero todavia un archivo unico integrado. Los archivos siguen separados
por fuente porque aun falta una modificacion manual adicional antes de unir, deduplicar y validar
la base final.

================================================================================
2. ACTIVIDADES REALIZADAS HASTA ESTE PUNTO
================================================================================

2.1 Conversion inicial a modelo canonico
---------------------------------------
Se convirtieron los archivos descargados desde las fuentes originales a un formato canonico
comun. Para cada fuente se revisaron las columnas disponibles y se mapearon solamente los
campos con evidencia directa en los datos.

Reglas aplicadas:
- No se inventaron DOI, ISBN, ISSN, afiliaciones, autores, keywords, abstracts ni subareas.
- Los DOI se normalizaron eliminando prefijos como https://doi.org/ y http://dx.doi.org/.
- El año se normalizo a entero cuando fue posible.
- La columna Subarea se dejo vacia por falta de evidencia suficientemente clara.
- Afiliacion2 se dejo vacia en la version canonica simplificada; la informacion institucional se
  concentro en Afiliacion1 cuando la fuente lo permitia.
- Solo se permitieron las areas: ISBD, CC, IA, TC, SIAV y RS.

2.2 Revision posterior de Area
------------------------------
Despues de la conversion canonica, se revisaron los 8 archivos por fuente para reasignar la
columna Area con base en el contenido de cada articulo.

Criterios usados para clasificar:
- Titulo
- Keywords
- Abstract
- Evidencia tematica contenida en el propio archivo canonico

No se hicieron busquedas en internet para esta revision.
No se modificaron metadatos bibliograficos distintos de Area y Subarea.
Subarea quedo vacia en todos los registros.
No se eliminaron registros por NO_CS, porque no se identificaron articulos 100% ajenos a
Ciencias de la Computacion con la informacion disponible.

8. CRITERIOS DE CLASIFICACION UTILIZADOS
================================================================================

ISBD - Ingenieria de Software, Bases de Datos y Sistemas de Informacion
-----------------------------------------------------------------------
Se uso ISBD para articulos cuyo aporte principal fue ingenieria de software, arquitectura de
software, sistemas de informacion, APIs, bibliotecas, herramientas web, HCI, UX, interfaces,
CSCW, plataformas interactivas, medicion de software, program synthesis aplicado a software o
sistemas de apoyo a usuarios.

CC - Computacion Cientifica
---------------------------
Se uso CC para articulos centrados en simulacion cientifica, modelado numerico, HPC,
GPU/CUDA, inversion numerica, solvers, PDE/ODE, FEM/FDM, computacion geofisica, astronomia
computacional, modelado fisico, software cientifico o pipelines de procesamiento cientifico.

IA - Inteligencia Artificial
----------------------------
Se uso IA para articulos centrados en aprendizaje automatico general, NLP, LLM, aprendizaje por
refuerzo, modelos generativos, robota inteligente, metaheuristicas, optimizacion evolutiva,
clasificacion, regresion, modelos predictivos o inteligencia computacional, salvo que el dominio
principal correspondiera claramente a señales/vision, seguridad, software, teoria o computacion
cientifica.

RS - Redes y Seguridad
----------------------
Se uso RS para articulos sobre privacidad, cifrado, esteganografia, watermarking, autenticacion,
blockchain, smart contracts, seguridad de datos, OpenPGP, IoT, fog/edge computing, redes,
protocolos, DoS/DDoS, deteccion de anomalias de infraestructura o sistemas distribuidos con
foco de seguridad/redes.

TC - Teoria de la Computacion
-----------------------------
Se uso TC para articulos de logica, verificacion formal, computabilidad distribuida, consenso,
linearizability, wait-freedom, estructuras concurrentes, teoria de grafos, geometria computacional,
automatas, complejidad, algoritmos teoricos, campos finitos, codigos y problemas formales.

SIAV - Señales, Imagenes y Ambientes Virtuales
----------------------------------------------
Se uso SIAV para articulos centrados en imagen, video, vision computacional, segmentacion,
reconstruccion, señales EEG/ECG/EMG/ERG/audio/voz/WiFi-CSI, wavelets, sensores, realidad
virtual, realidad aumentada, XR, interaccion haptica, ambientes inmersivos y analisis de señales
multimodales.
