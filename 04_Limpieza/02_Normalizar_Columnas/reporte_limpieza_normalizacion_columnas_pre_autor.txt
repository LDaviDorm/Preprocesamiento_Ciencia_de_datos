No se detectaron años fuera del rango esperado en la base procesada.

------------------------------------------------------------
5.4 ISBN
------------------------------------------------------------

Esta columna fue uno de los puntos corregidos con mayor cuidado.

Reglas aplicadas:

1. Tratar ISBN como texto.
2. Eliminar etiquetas como ISBN:, ISBN-10: o ISBN-13:.
3. No permitir notación científica.
4. Validar ISBN-10 e ISBN-13 cuando fue posible.
5. Corregir ISBN compactos válidos.
6. Mejorar guionización práctica de ISBN presentes en la base.
7. Separar múltiples ISBN con coma y espacio.
8. No inventar ISBN si el dato venía irrecuperable.

Motivo de la corrección:

En versiones previas, algunos ISBN quedaban compactos o con guiones incompletos, por ejemplo:

9781510673144
978-151067314-4
978-303147639-6

La versión corregida buscó evitar estos problemas y dejar ISBN más consistentes antes de cualquier deduplicación.

------------------------------------------------------------
5.5 ISSN
------------------------------------------------------------

Reglas aplicadas:

1. Quitar etiquetas como ISSN:, eISSN:, print ISSN u online ISSN.
2. Normalizar a formato ####-####.
3. Permitir X final.
4. Separar múltiples ISSN con coma y espacio.
5. Eliminar o dejar vacío si no se pudo extraer un ISSN válido.

------------------------------------------------------------
5.6 Doi
------------------------------------------------------------

Reglas aplicadas:

1. Normalizar DOI a minúsculas.
2. Eliminar prefijos como:

   https://doi.org/
   http://dx.doi.org/
   https://dx.doi.org/
   doi.org/
   doi:
   doi/

3. Conservar sólo DOI con estructura iniciada en 10.
4. Si había varios DOI, conservar el primero cuando era claramente identificable.

Resultado:

DOI no vacíos que no empiezan con 10.: 0

5.7 URL

5.8 Keywords

2. Usar coma y espacio como separador único.
3. Convertir separadores como punto y coma, barras verticales y saltos de línea a coma.
4. Quitar etiquetas editoriales como:

   index terms:
   ccs concepts:

5. Quitar HTML.
6. Quitar URLs.
7. Quitar basura editorial.
8. Deduplicar keywords preservando orden.

6. Quitar frases finales de disponibilidad de código, repositorios o datasets cuando contaminaban el abstract.

------------------------------------------------------------
7.1 Correcciones en Titulo
------------------------------------------------------------

También se limpiaron residuos puntuales de LaTeX o artefactos de extracción cuando afectaban títulos.

------------------------------------------------------------
7.2 Correcciones en Keywords
------------------------------------------------------------

Se aplicó una deduplicación suave para detectar keywords equivalentes aunque tuvieran guiones, puntuación o acentos diferentes.

Ejemplos de equivalencias tratadas como duplicadas:

bounded size registers / bounded-size registers
unbounded size registers / unbounded-size registers
gamma rays / gamma-rays

La clave de deduplicación se basó en:

1. quitar acentos;
2. pasar a minúsculas;
3. tratar guiones y puntuación como espacios;
4. colapsar espacios;
5. preservar el primer término observado.

------------------------------------------------------------
7.3 Correcciones en Abstract
------------------------------------------------------------

Se eliminaron residuos que todavía aparecían después de la normalización principal:

1. Copyright residual.
2. Frases legales finales.
3. Frases de disponibilidad de código o datos.
4. URLs finales de repositorios, datasets o páginas de proyecto.
5. Bloques tipo [Display omitted].
6. Bullets de highlights.
7. Mojibake común.
8. Residuos LaTeX como:

   mathsf
   mathrm
   mathbf
   frac
   varepsilon
   Theta left(...)
   Omega left(...)

El objetivo fue conservar el contenido académico del abstract, pero retirar elementos editoriales, legales o de extracción automática.
