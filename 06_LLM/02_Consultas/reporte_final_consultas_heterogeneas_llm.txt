2. Objetivo de esta etapa

El objetivo fue implementar una libreta capaz de responder preguntas descriptivas y predictivas sobre la base bibliográfica curada usando consultas heterogéneas.

La solución integra tres fuentes de consulta locales:

- SQLite:
  Para consultas tabulares sobre la base canónica artículo-autor.

- NetworkX:
  Para consultar el grafo principal y el subgrafo de coautoría.

- TF-IDF:
  Para similitud textual o temática usando Titulo, Keywords y Abstract.

El LLM local se usó como capa de redacción final y explicación de resultados, no como fuente de datos.

7. Control de calidad aplicado al prompt

Se corrigió el prompt para reducir interpretaciones indebidas del LLM.

Las reglas principales incorporadas fueron:

- Usar los códigos de área exactamente como aparecen en la base:
  ISBD, CC, IA, TC, SIAV, RS.

- No expandir ni interpretar el significado de las siglas de área.

- No inventar autores, artículos, afiliaciones, DOI, URL, áreas, keywords ni cantidades.

- No calcular totales nuevos si no aparecen explícitamente en la evidencia.

- Presentar las preguntas predictivas como resultados exploratorios o candidatos preliminares, no como predicciones garantizadas.

- Distinguir entre:
  TF-IDF = similitud textual o temática.
  Subgrafo de coautoría = cercanía estructural, vecinos comunes, grado o Jaccard.

- Incluir una nota de alcance cuando el resultado sea top-k, preliminar o exploratorio.
