REPORTE DE GENERACIÓN DE GRAFOS PARA CONSULTAS CON LLM

================================================================================
1. OBJETIVO DE LA ETAPA
================================================================================

El objetivo de esta etapa fue transformar la base canónica limpia y curada en una
representación de grafos que pudiera ser utilizada posteriormente como fuente de
información para consultas heterogéneas con LLM.

La construcción se hizo tomando como base el modelo de datos ya depurado, sin
modificar el contenido bibliográfico ni aplicar nuevas reglas de limpieza. En esta
fase no se eliminaron artículos, no se eliminaron autores y no se modificaron las
columnas canónicas. El propósito fue únicamente crear estructuras de consulta a
partir de la base final.

A diferencia de una versión filtrada para link prediction, en esta etapa se decidió
conservar toda la información disponible de la base. Por ello, los grafos generados
no aplican el filtro de autores con más de tres artículos. Esto permite que el LLM
pueda consultar autores con baja producción, artículos aislados, keywords poco
frecuentes y afiliaciones con pocos registros.

================================================================================
5. LÓGICA GENERAL DEL CÓDIGO
================================================================================

El código se organizó de forma sencilla y práctica con las siguientes etapas:

1. Configuración de rutas
   Se define la raíz del proyecto, el archivo de entrada y la carpeta de salida.
   También se incluye un fallback para pruebas fuera de la computadora local.

2. Validación de columnas
   Se comprueba que la base contenga exactamente las 14 columnas canónicas.
   Si faltan columnas o aparecen columnas extra, el código detiene la ejecución.

3. Limpieza auxiliar de valores
   Se usan funciones simples para tratar valores vacíos, quitar espacios dobles y
   evitar cadenas como nan, none, null o n/a.

4. Separación de keywords
   Las keywords se separan a partir de coma, punto y coma o pipe. Se pasan a
   minúsculas y se limpian signos residuales como punto y dos puntos. Esta
   operación sólo se usa para construir nodos keyword; no modifica la base original.

5. Creación del catálogo de autores
   Se agrupa por Autor_norm y se consolidan afiliaciones observadas por autor.
   También se calcula el número de artículos únicos por autor.

6. Construcción del grafo principal
   Se crea un grafo no dirigido con NetworkX. Los nodos se separan por tipo:
   artículo, keyword, autor y afiliación. Las aristas conectan:
   - autor con artículo
   - artículo con keyword
   - autor con afiliación

7. Construcción del subgrafo de coautoría
   Se agrupa la base por artículo. Para cada artículo, se generan todas las
   combinaciones de pares de autores y se crea una arista entre ellos. Si dos
   autores colaboraron en más de un artículo, la arista incrementa su peso.

8. Exportación
   Se guardan el grafo principal y el subgrafo en formato GML, compatible con
   NetworkX y otras herramientas de análisis de grafos.

9. Visualización
   Se generan visualizaciones de muestra para revisar que los grafos se hayan
   creado correctamente sin intentar mostrar todos los nodos al mismo tiempo.

10. Resumen
    Se genera un archivo JSON con métricas principales de la base y los grafos.

================================================================================
6. DISEÑO DEL GRAFO PRINCIPAL
================================================================================

El grafo principal se diseñó como una red heterogénea con cuatro tipos de nodos:

1. Artículo
   Representa cada artículo único de la base. Se identifica con el índice y el
   título. Los atributos del nodo artículo son:
   - nombre
   - indice
   - Year
   - ISBN
   - ISSN
   - Doi
   - URL
   - Area
   - Abstract

2. Autor
   Representa cada autor único de la base, usando Autor_norm.

3. Keyword
   Representa cada keyword única obtenida desde la columna Keywords.

4. Afiliación
   Representa cada entidad o dependencia presente en Afiliacion1 o Afiliacion2.

Tipos de aristas del grafo principal:

- autor_articulo: conecta un autor con un artículo en el que participa.
- articulo_keyword: conecta un artículo con cada keyword asociada.
- autor_afiliacion: conecta un autor con su afiliación o afiliaciones.

Para evitar colisiones entre nodos que pudieran tener el mismo texto, el código
usa identificadores internos con prefijo de tipo. Por ejemplo:

- articulo::indice::titulo
- autor::nombre
- keyword::termino
- afiliacion::dependencia

Esto evita que una keyword, una afiliación o un autor con textos similares se
mezclen por accidente.

Métricas del grafo principal:

- Nodos totales: 3,896
- Aristas totales: 5,078

Distribución de nodos por tipo:

- Artículos: 406
- Keywords: 2,889
- Autores: 550
- Afiliaciones: 51

La distribución muestra que las keywords son el tipo de nodo más numeroso. Esto
es esperable, porque un mismo artículo puede contener varios términos temáticos,
y muchas keywords aparecen sólo en pocos artículos.

================================================================================
7. DISEÑO DEL SUBGRAFO DE COAUTORÍA
================================================================================

El subgrafo de coautoría se creó para estudiar relaciones directas entre autores.
A diferencia del grafo principal, este subgrafo sólo contiene autores como nodos.

Regla de construcción:

- Se agrupa la base por artículo.
- Para cada artículo se obtiene la lista de autores.
- Cada par de autores dentro del mismo artículo recibe una arista.
- Si un par colabora en más de un artículo, se incrementa el peso de la arista.

Atributos de los nodos autor:

- nombre
- afiliacion1
- afiliacion2
- num_articulos

Atributos de las aristas:

- weight: número de artículos compartidos por el par de autores.
- articulos: títulos de los artículos en los que colaboraron.

Métricas del subgrafo:

- Nodos autor: 550
- Aristas de coautoría: 827
- Componentes conexas: 173
- Tamaño de la componente mayor: 60 autores

Estas métricas indican que la red de coautoría no forma un único componente
gigante. Existen múltiples grupos de colaboración, algunos pequeños y otros más
conectados. La componente mayor contiene 60 autores.
